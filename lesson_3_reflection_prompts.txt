When would you want to use a remote repository rather than keeping all your work
local?

    When multiple users will want to be able to view and or edit the code and 
    track changes.

Why might you want to always pull changes manually rather than having Git
automatically stay up-to-date with your remote repository?

    Manually pulling/pushing commits rather than doing it automatically will
    protect the version available on git hub from any bugs or features in a 
    local commit that are not yet ready for production.
